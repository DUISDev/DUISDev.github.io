# DUIS Dev - Blog

Статический блог с простой админ-панелью и Java-сборщиком, который собирает главную страницу из файлов постов.

## Что умеет проект

- Показывает посты на главной странице `index.html`.
- Хранит посты как отдельные HTML-файлы в папке `posts`.
- Генерирует список постов через Java-сборщик `build-index.java`.
- Поддерживает медиа в постах: изображения, видео, аудио и документы.
- Для документов на главной создаёт карточку с кнопкой скачивания.
- Поддерживает базовое форматирование текста в стиле WhatsApp:
  - `*жирный*`
  - `_курсив_`
  - `<код>`
  - `~зачёркнутый~`

## Структура проекта

```text
.
├─ index.html              # Главная страница (генерируется)
├─ build-index.java        # Сборщик главной страницы
├─ admin/
│  └─ index.html           # Админка для создания постов
├─ posts/                  # Файлы постов (post1.html, post2.html, ...)
├─ assets/                 # Картинки, видео, аудио, документы
├─ css/
│  └─ style.css
└─ js/
   └─ script.js
```

## Требования

- Java 11+ (рекомендуется Java 17+).
- Любой статический HTTP-сервер для просмотра сайта.

## Как запустить

1. Открой проект в терминале.
2. Скомпилируй сборщик:

```bash
javac build-index.java
```

3. Запусти сборку:

```bash
java BlogIndexGenerator .
```

4. Открой `index.html` через локальный HTTP-сервер.

## Как добавить пост

Есть два варианта:

- Через `admin/index.html` (форма создаёт готовый HTML поста).
- Вручную: создать файл `posts/postN.html`.

Важно:

- Для документов и медиа используйте пути вида `assets/имя_файла.ext`.
- Файлы должны реально лежать в корневой папке `assets`.

## Формат поста (пример)

```html
<article class="post-preview">
    <img src="assets/example.png" alt="Пример">
    <h2><a>Заголовок поста</a></h2><br>
    <p>Текст поста с *жирным*, _курсивом_, <кодом> и ~зачёркнутым~.</p><br>
    <p class="post-date">28 апреля 2026 г.</p><br>
</article>
```

## Как работает сборщик

`build-index.java`:

- Берёт файлы `posts/post*.html`.
- Сортирует их по номеру (новые выше).
- Применяет текстовое форматирование (`* _ < > ~`) к содержимому.
- Определяет тип медиа:
  - изображение/видео/аудио остаются обычной карточкой;
  - документ превращается в карточку с кнопкой `Скачать`.
- Пересобирает `index.html`.

## Полезно знать

- После изменений в постах всегда перезапускай сборщик.
- Если изменения не видны в браузере — обнови страницу с очисткой кэша (`Ctrl+F5`).
- Если скачивание документа не работает, сначала проверь:
  - корректность пути `assets/...` в посте;
  - наличие файла в папке `assets`.
